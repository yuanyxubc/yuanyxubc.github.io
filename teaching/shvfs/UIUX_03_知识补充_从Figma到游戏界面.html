Game UI Pipeline3 / 28
Figma × Unreal

Figma 和 Unreal（UMG）是什么关系

Figma

  • 做设计与沟通用的工具
  • 产出视觉稿、交互流程图、设计规范
  • 不是游戏里能跑的东西
  • 面向策划 / 制作人 / 团队评审
  • 迭代快、门槛低、适合早期探索

Unreal · UMG

  • 做真正运行的界面的地方
  • Widget Blueprint 拖控件、拉锚点、接数据
  • 要考虑分辨率、性能、蓝图逻辑
  • 面向 UI 程序 / 技术美术
  • 门槛较高，是最终落地实现
Game UI Pipeline4 / 28
Figma × Unreal

为什么不干脆直接在 UMG 里做

迭代速度

改一版几秒 vs 几十秒

Figma 改颜色/间距即时生效；UMG 改动常要重新编译、热重载。

评审沟通

谁都能打开看

发个链接策划/制作人就能评论；没人会为了看稿单独装引擎。

早期探索

先想清楚再盖房子

多方案对比阶段不需要"真实"，Figma 几分钟出好几版。

技能门槛

视觉设计师也能上手

不需要先学蓝图、锚点这些引擎概念。

组件系统

设计规范统一维护

Figma Component + Auto Layout 管理设计系统非常成熟。

分工协作

各司其职

视觉与实现拆开，才能让更多角色并行工作、互不阻塞。

Figma × Unreal

从 Figma 到 UMG，中间发生了什么

1
设计稿 + 规范Figma 里定好布局、样式、交互流程 — 这是"图纸"，还不是能跑的东西
2
资源导出图标、九宫格边框等导出成 PNG/贴图；文字/颜色/间距整理成规范文档
3
在 UMG 里重新搭建不是"导入"，是照图纸重新盖房子：容器、锚点、自适应都要重做
4
动效与交互重做Animation 面板重做关键帧动效，蓝图连线实现点击等交互逻辑

插件（如 Figma2UMG）可以自动生成部分 Widget 结构，省一些体力活，但复杂动效和数据绑定仍需人工完成。

Game UI Pipeline6 / 28
Figma × Unreal · 实操建议

Figma 阶段：该做 / 该避免

✓ 该做

  • 用 Auto Layout 搭模块，对应 UMG 的 Horizontal/Vertical Box
  • 标注每个元素的锚点意图（贴左上/居中/贴右下）
  • 九宫格素材预留足够的安全边角
  • 画布叠加安全区参考线，核心元素不贴边
  • 统一目标分辨率作为设计基准
  • 图层/组件规范命名（Button/PlayBtn 而不是 Rectangle 23）

✗ 该避免

  • 用 UMG 还原不了的复杂特效（多层混合阴影、叠加模糊）
  • 图层随意命名，导致接手/转换成本暴涨
  • 不考虑分辨率变化的纯自由拖拽定位
  • 九宫格素材没留边距，拉伸就变形
  • 多套基准分辨率混用，团队标尺对不齐
  • 把 Smart Animate 当成最终动效标准来交付
Game UI Pipeline7 / 28
Figma × Unreal · 进阶

如果想自己动手做 UMG，还要补充学什么

蓝图基础

看懂并能搭简单 Event Graph，做点击跳转这类基础交互逻辑。

UMG 容器逻辑

Canvas Panel、Horizontal/Vertical Box、Size Box、Scale Box 的区别与使用场景。

锚点系统实操

0–1 坐标系统、Anchor 拖拽方式，以及在不同分辨率下的实测适配。

素材接入规范

Draw As Box 九宫格设置、图片 Brush 的导入与替换流程。

动画面板基础

关键帧动画怎么做，Fade / Scale / Move 这类常见动效怎么搭。

版本控制 + 性能意识

Perforce/Git 日常操作避免冲突；知道控件层级不能太深、了解 Invalidation Box 大致用途。

Studio Structures

团队规模如何影响分工

🧑‍🤝‍🧑

小型 / 独立团队

示意：约 5–15 人
  • 1 人身兼 UI/UX Designer + UI Artist，甚至部分 UMG 实现
  • 1 名 Generalist Programmer 同时做 UI 和 Gameplay 逻辑
  • 几乎没有专职 Technical Artist；一人多岗，交接成本低但个人工作量大
🏢

中型团队

示意：约 30–80 人
  • 1–2 名 UI/UX Designer + 1 名 UI Artist
  • 1 名 Technical Artist（常兼 UI Programmer 部分工作）
  • 2–3 名 Gameplay Programmer 分摊给多个系统共用
🏙️

大型 / 3A 团队

示意：300 人以上
  • UI 团队本身可达 10–20+ 人
  • 细分为 UX Research / UI Design / UI Art / UI Engineering / Technical Art 等小组
  • 各小组有 Lead，还有专职 UI QA、专职手柄导航或本地化排版负责人
以上人数与架构为行业常见模式的示意举例，用来说明规模效应，并非某个具体公司的真实组织架构，实际会因项目类型、发行阶段等因素差异很大。

Interview · UI/UX Designer

UI/UX 岗位面试考什么

01

作品集深挖

讲清楚一个项目从需求到落地的完整决策过程，playtest 后做了哪些调整。

02

现场设计挑战

限时设计一个界面（如商店/背包），当场讲思路和取舍。

03

工具熟练度

Figma 组件、Auto Layout、大型团队文件结构管理能力。

04

UX 原理应用

能否把 Fitts's Law、Hick's Law 等原理用游戏场景讲明白。

05

平台适配思维

同一界面在手柄/触屏/鼠标下该如何调整。

06

协作与沟通

如何应对和程序/制作人的意见分歧、如何处理裁剪需求。

常见真题举例

设计挑战给你 30 分钟，为一款手游设计一个抽卡/商店界面，说说你的取舍。
批判分析这是某款游戏的 HUD 截图，你会改哪里？为什么？
原理应用举一个你用过的 UX 原理，说说它怎么影响了你的设计决定。
无障碍你怎么为色盲玩家 / 单手操作玩家设计界面？
行为面试说一次你和工程师/制作人在界面方案上有分歧，最后怎么解决的。
Game UI Pipeline20 / 28
Technical Artist

技术美术要学什么

编程基础

Python（工具/管线脚本）、C++/C# 基础、Unreal Blueprint 深度使用。

引擎 UI 系统

UMG 的 Anchor、Invalidation Box、Retainer Box；Unity 的 UI Toolkit / uGUI。

Shader / 材质

HLSL、材质编辑器，做发光/溶解/扫光等 UI 特效，理解 Render Target。

性能优化

Draw Call 合批、Overdraw 分析、用 Unreal Insights / Profiler 排查卡顿。

工具开发

写 Editor 工具/脚本自动化重复劳动，比如批量导入 Figma 资源。

跨部门沟通

把设计师的意图翻译成技术约束，也把技术限制解释给设计师听。

技术美术面试怎么考

Written Test

笔试题一般考什么

UI/UX Designer 笔试

  • 设计理论基础：色彩、字体排版、格式塔原理、栅格系统
  • 游戏行业与竞品认知：同类型产品的 UI 趋势
  • 案例分析/找茬：给界面截图找可用性、层级问题
  • 现场排版/线框题：给素材现场搭一版界面
  • 交互流程图：画出某功能的用户流程
  • 文案题：写错误提示、引导语等微交互文案

Technical Artist 笔试

  • 编程基础：Python / C++ 语法与简单逻辑题
  • 数学基础：向量、矩阵、三角函数
  • Shader / 材质概念题：解释某视觉效果的实现思路
  • 引擎机制问答：UMG / uGUI 底层怎么工作
  • 找问题题：给一段有性能隐患的代码/蓝图找瓶颈

Campus vs Experienced Hire

校招 vs 社招：对应 Junior / Senior Artist

校招 → Junior / Associate Artist

  • 面向应届生或毕业 1-2 年内
  • 看重基础功底、学习能力、可塑性
  • 作品集重点看思考过程，不强调实际项目经验
  • 通常配 Mentor 带教，薪资较低，晋升路径明确
  • 不要求独立带过完整项目

社招 → Senior（甚至 Lead/Principal）Artist

  • 通常要求 2 年以上，Senior 一般 3-5 年以上
  • 看重实际上线项目经验、独立负责模块的能力
  • 对管线理解深度、带新人/跨部门推动能力
  • 要求理解引擎实现限制，不只是"画得好看"
  • "管线适配性"在这一阶段被看得更重
Game UI Pipeline24 / 28
Applying to Schools

申请欧美/英国设计院校：UI/UX 作品集要求

提交形式

通常 8-20 个项目，PDF 或网站链接提交；可交互原型可发可点击链接，很多学校不强制要求代码或特定软件背景。

考核重点

草图/线框/迭代过程比最终图更重要；很多学校要求每个作品配一段反思短文，讲清楚设计决策。

UI/UX 要做到的程度

能画信息架构和用户流程图；能做出中低保真可点击原型；有基础排版和层级意识，不要求美术功底拔尖。

为什么 Figma 重要

版本迭代一目了然；不用写代码就能做可交互原型；也是入学后默认的协作工具，提前熟悉能把精力放在设计本身。

Applying to Schools

游戏设计 vs 游戏交互（UX）专业：作品集差异

游戏设计专业

  • 更看重游戏概念与设计能力，不强调视觉打磨
  • 常见形式：游戏概念提案（题材/玩法机制图/关卡角色大纲/世界地图）或代码作品集（源码片段+说明）
  • VFS 甚至允许用 Foundation 先修项目替代作品集
  • 竞争激烈的项目（如 Breda）作品集之外还有面试，讨论创作过程

游戏交互 / UX 设计方向

  • 更接近通用 UI/UX 作品集要求：线框图、可点击原型、用户研究记录
  • 核心考核设计思维过程，而不是画面好不好看
  • 需要展示对游戏专属交互场景的理解：HUD、菜单、新手引导、手柄导航
  • Figma 或类似原型工具几乎是标配，能直接展示可玩的界面流程
以上以 Vancouver Film School、Abertay University、Breda University of Applied Sciences 等游戏设计相关专业公开的申请要求为参考示例，具体要求以各校官网当年公布为准。

Takeaways

给 VFS 同学的建议

✓
作品集要体现"决策过程"，不只是好看的图为什么这么设计、改过几版、playtest 反馈是什么
✓
至少熟悉一个引擎的 UI 系统UMG 或 Unity UI Toolkit，能证明你懂"设计要怎么落地"
✓
了解你上下游角色在做什么即使你只想做 UX，懂一点 Technical Artist 的限制会让沟通更顺
✓
校内项目就练习"多人协作式"分工模拟真实工作室里设计-美术-程序的交接流程
★
公司里这是好几个人分工做的，但求职时要展现"管线适配性"真实工作中角色是分开的，但面试官更青睐"一专多能"（T-shaped）的候选人——尤其中小型工作室，会更看重你对整个管线（设计→实现）的理解广度，而不只是单一技能
