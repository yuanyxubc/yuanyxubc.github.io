课程目标

本课程从游戏界面案例出发，学习如何观察 UI/UX 问题、拆解信息层级、使用 Figma 组织界面，并清楚说明自己的设计判断。

Final Project 将以 Figma 项目板形式提交，包含案例分析、流程结构、关键界面、设计说明与课堂展示。

Final Project Checklist

  • 1 个游戏模块案例分析
  • 1 张流程图或信息结构图
  • 恰好 6 个 Figma 高保真界面：战斗 HUD、人物信息、背包 / 装备、技能树、主菜单、设置
  • 至少 8 个课程组件的使用或改造
  • 5 分钟课堂展示

Final Project：六界面高保真 Figma 项目板

为同一个类暗黑式 Action RPG 建立一套高保真 UI 系统。最终必须提交恰好六张界面：战斗 HUD、人物信息、背包 / 装备、技能树、主菜单、设置。六张必须共享同一套视觉方向、语义 Variables、Components 与 Variants；不能把六张做成六种互不相关的风格。

固定作业范围

战斗 HUD 人物信息 背包 / 装备 技能树 主菜单 设置

六张界面分别要做到什么

界面最低内容与关键状态本课系统要求
01 · 战斗 HUD生命 / 资源、当前目标、技能冷却、危险提示和操作反馈；至少展示正常与危险状态。使用 Constraints 固定 HUD 锚点；技能槽和状态提示来自 Components / Variants。
02 · 人物信息基础属性、抗性、装备摘要、升级或加点入口；数值变化要能比较。属性行使用 Auto Layout；数值、标题、Surface 引用语义 Variables。
03 · 背包 / 装备物品格、装备槽、Tooltip、装备比较、已装备 / 新物品 / 不可装备状态。这张先做 Benchmark；综合使用 Variables、Components、Variants 与 Auto Layout。
04 · 技能树锁定、可学习、悬停、已学习、满级，并说明前置条件和资源消耗。技能节点必须是 Component Set；用 Variant 属性切换 State。
05 · 主菜单继续、人物、背包、技能、设置、退出；显示默认、Hover、Focus / Selected。按钮和导航复用同一组件；列表使用 Auto Layout。
06 · 设置Toggle、Slider、Keybind；包含 Hover / Focus、未保存、按键冲突与确认。控件使用 Variants；Danger / Success 与间距全部引用 Variables。

Tooltip、装备比较、掉落提示、未保存条和按键冲突提示属于上述界面的状态或覆盖层，不另算第七张。

推荐制作顺序

背包 / 装备 Benchmark → 人物信息 + 技能树 → 战斗 HUD → 主菜单 + 设置 → 六页一致性检查。第一张先锁定字体、材质、Token、组件尺寸和状态语言；其余五张复用，不重新探索风格。

评分标准

Final Project 将根据以下标准评分。

项目占比评分依据
案例分析20%是否能明确指出原界面的 UX 问题；截图标注是否清楚；是否使用 3C 或信息层级语言。
流程与结构20%流程图是否完整；玩家下一步是否明确；是否避免不必要的步骤。
界面完成度25%六张规定高保真是否全部完成；1920 × 1080 下是否清楚；组件、Variables 与状态是否一致。
设计说明20%能否解释自己的设计选择；是否能说清楚每页解决什么问题。
Final Presentation15%展示结构是否清楚；能否在 5 分钟内说明问题、方案和取舍。

可选加分：完成基础点击跳转、补充更多状态动效或可访问性验证。加分内容不能代替六张规定高保真。
