# UIUX Week2 FigJam 图版提示词（中英双语）

把下面这段复制到 FigJam AI / Figma Make FigJam board 里使用。提示词要求图版上所有关键任务都同时出现英文和中文，方便学生看到英文标签时也知道要做什么。

```text
Create a bilingual pixel-art themed FigJam board for a Week2 UI/UX class.
为 UI/UX 第二周课程创建一个中英双语、像素风 FigJam 图版。

Board title / 图版标题:
UIUX WEEK 02 · Reference 参考 → Analyze 分析 → Imitate 临摹 → Explain 解释

Visual style / 视觉风格:
Dark pixel game UI, deep navy background, chunky 8-bit borders, high-contrast section headers, small pixel squares as icons, no soft gradients.
深蓝像素游戏界面风格，8-bit 粗边框，高对比标题，小像素方块图标，不使用柔和渐变。文字必须清楚、可读，五个学生能同时操作而不重叠。

Add a clear instruction card near the top-left / 在左上角添加清楚的说明卡:
Title / 标题: How to use this board · 这个图版怎么用
Text / 正文:
1. Class flow board: follow the zones from Reference to Feedback.
1. 课堂流程板：按 Reference 参考 → Analysis 分析 → Figma 链接 → Feedback 反馈 的顺序完成。
2. Week2 submission entry: teacher checks your own Player lane at the end.
2. 第二周提交入口：下课前老师检查每个人自己的 Player 区域。
3. Teacher check map: empty zones show where you are stuck.
3. 老师巡场检查表：哪一格空着，就说明你卡在那一步。

Create five vertical player lanes / 创建五个纵向学生区域:
Player1: Pascal
Player2: Joe
Player3: Hank
Player4: Andrew
Player5: Allan

Each player lane must contain four stacked bilingual zones / 每个学生区域包含四个中英双语区块:

1. REFERENCE / 参考
Title / 标题: Competitor / Case Study Page · 竞品 / 作品集分析页
Prompt text / 提示文字:
Paste one UI/UX case study, competitor analysis page, or portfolio page.
贴一页 UI/UX case study、竞品分析页或作品集页面。
Write / 写一句:
I want to learn this page's ______.
我想学这页的 ______。
Small placeholders / 小占位:
- Link / screenshot · 链接 / 截图
- What structure can I copy? · 我可以借鉴什么结构？

2. ANALYSIS / 分析
Title / 标题: One Game Screen Analysis · 一页游戏界面分析
Prompt text / 提示文字:
Paste one real game UI screenshot.
贴一张真实游戏 UI 截图。
Add 3 annotations using this format:
用下面格式添加 3 个标注:
I see ______. / 我看到 ______。
It works / fails because ______. / 它有效或有问题，因为 ______。
Reusable rule: ______. / 可复用规则：______。
Small placeholders / 小占位:
- Screenshot · 截图
- Annotation 1 · 标注 1
- Annotation 2 · 标注 2
- Annotation 3 · 标注 3
- One reusable design rule · 一条可复用设计规则

3. FIGMA / 链接
Title / 标题: Trace / Original Figma Link · 临摹 / 原创 Figma 链接
Prompt text / 提示文字:
Paste your personal Figma link.
贴你自己的个人 Figma 文件链接。
Choose one:
二选一:
Trace a local component from your analysis.
从你的分析页里圈出的局部组件开始临摹。
Or make an original component for the MBTI slime project.
或者为 MBTI 史莱姆项目做一个原创组件。
Small placeholders / 小占位:
- Figma link · Figma 链接
- What did I copy / transform? · 我临摹 / 转化了什么？
- What component did I work on? · 我做了哪个组件？

4. FEEDBACK / 反馈
Title / 标题: Peer Notes · 同伴反馈
Prompt text / 提示文字:
Classmates add two notes:
同学添加两条反馈:
Green note: I understood ______.
绿色便签：我看懂了 ______。
Pink note: I still wonder ______.
粉色便签：我还想问 ______。
Small placeholders / 小占位:
- Green peer note · 绿色同伴反馈
- Pink peer note · 粉色同伴问题

Add a shared class area on the right / 在右侧添加共享课堂区域:

SHARED REFERENCES / 共享参考
Purpose / 用途:
Put links that the whole class can reuse.
放全班都可以复用的参考链接。
- Useful game UI links · 有用的游戏 UI 链接
- Useful portfolio / case study links · 有用的作品集 / 案例分析链接
- Game UI Database
- Interface In Game

COMPONENT IDEAS / 组件想法
Purpose / 用途:
Collect components that are small enough to trace or transform today.
收集今天适合临摹或转化的小组件。
- Dialogue bubble · 对话气泡
- Choice button group · 选择按钮组
- Progress bar · 进度条
- MBTI offset panel · MBTI 偏移面板
- Result card · 结果卡片
- Right-click menu · 右键菜单

PARKING LOT / 问题停车场
Purpose / 用途:
When students are stuck, they should post the question here first instead of stopping.
学生卡住时，先把问题贴到这里，不要停在原地。
- Questions for teacher · 给老师的问题
- Things to fix later · 之后要修的东西
- Good techniques to reuse · 值得复用的技巧

Add timer cards at the top / 顶部添加计时卡:
5 min · Find your lane · 找到自己的区域
10 min · Find case page · 找竞品 / 案例分析页
15 min · One screen analysis · 做一页游戏界面分析
7 min · Peer check · 同伴检查
20 min · Figma trace / original · Figma 临摹 / 原创
5 min · Final FigJam check · 最终图版检查

Use color coding / 使用颜色编码:
Yellow = Reference / 参考
Cyan = Analysis / 分析
Green = Reusable rule / 可复用规则
Purple = Figma link / Figma 链接
Pink = Peer question / 同伴问题

Make the board clear enough that five students can work at the same time without overlapping.
确保图版足够清楚，五个学生可以同时操作，不会互相遮挡。
```
